説明

  • ソースコード -> gitのレポジトリにpushして下さい (最終日の時点で必ず一度 提出. その後も変更可)
  • 最終日発表 スライド 発表に先立ち, 2025年度「大規模ソフトウェアを手探る」チームリスト からリンクが生えているようにしてください. リンクはGoogle docsやOffice Online, Dropboxなどでも可. しかし, 保存を考えて，できればgithub に入れてください(別のレポジトリを作る)．
  • レポート スライド同様, 2025年度「大規模ソフトウェアを手探る」チームリスト からリンクが生えているようにしてください.
  • UTOL報告 形式的ですが記録のため, 各自がUTOLの 電気電子情報実験・演習第二 で報告してください. 感想や, heartwarmingな時候の挨拶などを歓迎します.
レポートの中身について: このスライドで 説明したとおり，blogなどにパブリックな記事として書いても良いし， 世間の人に読ませるのがどうしてもはばかられるという場合は， markdownを使って記述し，それをgithubから読めるようにするのでもよいです． 書きっぷり及びmarkdownファイルの例としては， こちらで用意したサンプルも参照． 我々への報告ではなく，世間の人に読ませるという体で， やったこと，そこへ至る過程なども整理．複数の記事に分けても可． 道半ばで時間切れの人も，後進に役に立つ記事を残せばよい． こういう環境で，こういうことをやろうとするとどんなエラーに遭遇し， どう回避したかを書く．
